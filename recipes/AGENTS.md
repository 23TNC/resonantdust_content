# content/recipes/

Recipe registry. Each recipe is a JSON **array of statement strings**
keyed by recipe id under a `<type>` or `<type>.<category>` outer key.
The shared content crate parses these into a `RecipeDef` registry
consumed by the SpacetimeDB shard and the PixiJS client (via the
wasm pkg).

## Layout

```
content/recipes/
  AGENTS.md            ← this file
  id.json              ← stable u10 ids per recipe key. Generated by gen-ids.py.
  types.json           ← (type, category) → numeric id mapping. Hand-edited.
  data/*.json          ← recipe definitions. Auto-discovered at build.
```

## File format

Each file under `data/` is a top-level object keyed by recipe type:

```jsonc
{
  "stack.up":   { "<recipe_id>": [<statements>], ... },
  "stack.down": { ... },
  "magnetic.up":{ ... },
  "on_create":  { ... }
}
```

Outer keys:

| Key             | Resolves to                                  |
| --------------- | -------------------------------------------- |
| `stack.up`      | `RecipeType::Stack(Up)`                       |
| `stack.down`    | `RecipeType::Stack(Down)`                     |
| `magnetic.up`   | `RecipeType::Magnetic(Up)`                    |
| `magnetic.down` | `RecipeType::Magnetic(Down)`                  |
| `on_create`     | `RecipeType::OnCreate` (collapses the `self` category — it's the only one) |

Multiple types per file is fine — the loader walks every top-level
key. Unknown keys (including the `_comment` JSON-doc convention) are
silently skipped so content can outpace the Rust enum.

Each recipe is a JSON array; every element is one **statement**.

## Statement grammar

```
statement   := path                       # verb-only
             | path ': ' value             # path + scalar
path        := segment ('.' segment)*
segment     := identifier | integer
identifier  := [a-z_][a-z0-9_]*           # ASCII lowercase + digits + _
integer     := decimal digits
value       := string | integer
```

The single `': '` (colon + space) is the only legal whitespace inside
a statement. Bare `:` is treated as path content — keeps the door
open for future colon-bearing tokens. Integer values are parsed
eagerly (so per-bucket grammars dispatch on type without re-parsing);
anything that isn't a valid `i64` becomes a string.

Path segments come from a **closed reserved-word set** —
identifiers that don't appear in the grammar's vocabulary will fail
at registry build. This is deliberate: it gives one dispatcher over
known tokens rather than per-arm validation. Aspect names (and any
other content-namespace identifiers) cannot collide with the
reserved set; see [Reserved words](#reserved-words) below.

## Reserved words

Path-grammar tokens that aspect names, flag names, and other
content-namespace identifiers must not shadow:

| Group | Tokens |
| --- | --- |
| Top-level buckets | `input`, `output`, `duration`, `style` |
| Anchors | `hex`, `root`, `slot`, `actor`, `has` |
| Resolvers | `owner`, `parent` |
| Match selectors | `def_id`, `aspect` |
| Predicates | `min` |
| Output sub-buckets | `create`, `destroy`, `modify` |
| Modify ops | `add`, `sub`, `set` |
| Locations | `inventory` |
| Tier markers | `default`, `when` |

`is_reserved_aspect_name(name)` in
[`content/src/recipe_statement.rs`](../src/recipe_statement.rs) is
the canonical check.

## Top-level buckets

Each statement's first path segment selects a bucket:

| Bucket | Purpose |
| --- | --- |
| `input` | Match-side preconditions — what cards / aspects / has-predicates the recipe binds. |
| `output` | Completion-side effects — `create`, `destroy`, `modify` sub-buckets. |
| `duration` | Action duration (fixed or conditional). |
| `style` | Client render hint (`none` / `ltr` / `rtl`). |

### Card refs

Most statements reference a card by **anchor** (and optional
resolvers). Anchors are recipe-relative — `hex`, `root`, `slot.<i>`,
`actor` (sugar for `slot.0`). Each can be followed by `.owner`
(inventory container) — the closed resolver set.

Examples:
- `hex` — the hex anchor (Card row or synthetic-hex tile).
- `slot.0` — slot 0, also addressable as `actor`.
- `slot.0.owner` — the actor's inventory container (typically a soul).
- `root` — the chain root.

`slot.<i>.owner` collapses semantically to `actor` for product
placement: under today's `ProductOwner::Actor` resolution, the
target is always `inventory_container_for_at(actor_id)`. v1 accepts
both spellings.

## `input` statements

```
input.<anchor>.def_id: <key>                # bare-card predicate
input.<anchor>.aspect.<name>.min: <int>     # aspect predicate (≥ min)
input.has.def_id: <key>                     # has-predicate (actor's UP stack)
input.has.aspect.<name>.min: <int>          # has-predicate via aspect
```

Valid anchors: `hex`, `root`, `slot.<i>`, `actor`. `has` is its own
top-level grammar entry (implicit `actor` role + UP-stack direction
for v1; `has.root.*` and `has_below.*` are deferred).

Each anchor accepts at most one predicate today (no tree-shaped
`and` / `or` / `not` — those reserved for a later grammar widening).

## `output` statements

Three sub-buckets:

```
output.destroy.<card_ref>
output.create.<card_ref>.<location_id>: <def_key>
output.modify.<card_ref>.aspect.<name>.<op>: <int>
```

### `destroy`

Flips `FLAG_DEAD` on the resolved card at `completion_ms`. v1 refs:
`root`, `hex`, `actor`, `slot.<i>`. Synthetic-hex destroy
(`output.destroy.hex` on a tile action) reverts the tile to its
biome via `world_gen::biome_for`.

### `create`

Spawns one card at the resolved location. v1 location: `inventory`
only — resolves to `(surface = INVENTORY_LAYER, macro_zone =
card_id, micro_zone = 0, micro_location = 0)` of the referenced
card.

v1 card refs for `create`:
- `root` → `ProductOwner::Root`
- `hex` → `ProductOwner::Hex`
- `actor` / `slot.0` / `slot.0.owner` → `ProductOwner::Actor`

Multi-spawn: repeat the statement with the same prefix.
`cut_tree`'s `output.create.slot.0.owner.inventory: corpus-` +
`output.create.slot.0.owner.inventory: log` spawns two cards at
the actor's container in order.

### `modify`

Mutates the row's stock slot for `<name>` on the resolved card.
v1 ref: **`hex` only** — only tile defs declare row-mutable stock
today, and only the hex anchor carries tile data. Card-rooted writes
fail at registry build until per-card stock storage lands.

Ops:

| Op | Semantics | Saturation |
| --- | --- | --- |
| `add` | `stock += delta` | clamps at `slot.max` (def-declared) |
| `sub` | `stock -= delta` | saturates at 0 |
| `set` | `stock = delta` | clamped to `[0, slot.max]` |

`delta` is a positive integer (`1..=3` for `add`/`sub`; `0..=3` for
`set` since slots are u2).

**Propose-time check.** When op is `sub`, `propose_action` rejects
if the tile's current stock is below the delta. `add` / `set`
saturate silently — over-fill / over-set don't refuse the action.

**Anchor-coupled gate.** When the recipe's `input.hex.aspect.<name>.min: M`
predicates on the same aspect a `sub: N` modifies, the parser
enforces `M >= N` so a recipe can never match a tile whose stock
can't satisfy its own sub.

## `duration` statements

```
duration.default: <seconds>
duration.when.aspect.<name>.min.<N>: <seconds>
```

- `duration.default` is required.
- `duration.when.aspect.<name>.min.<N>: <seconds>` adds a tier — at
  completion the matcher evaluates each `when` against the recipe's
  aspect pool (root + slots aggregation, today's behaviour) in
  registry order; the first match wins. If none match,
  `duration.default` is the fallback.

Example (`fleeting` recipe, on_create):

```jsonc
"fleeting": [
  "input.root.aspect.fleeting.min: 1",
  "output.destroy.root",
  "duration.when.aspect.fleeting.min.4: 20",
  "duration.when.aspect.fleeting.min.3: 15",
  "duration.when.aspect.fleeting.min.2: 10",
  "duration.default: 5",
  "style.default: rtl"
]
```

## `style` statements

```
style.default: <none|ltr|rtl>
```

Mapped to the `progress_style` field (3 bits) on the actor's
completion row. Future tiers (3..=7) slot in by extending the
enum; `style.default: <unknown>` errors.

## Run order at completion

`action_completion::apply` runs three passes against `recipe_def`:

1. **destroy** — flip `FLAG_DEAD` on every ref. Synthetic-hex
   destroy reverts the tile to its biome.
2. **modify** — apply stock mutations via `set_tile_stock_at`.
3. **create** — spawn cards via `cards::create_at`.

Destroy runs before modify so a modify reading the same hex sees
the pre-revert tile def; create runs last so spawned cards don't
collide with destroyed refs.

## Worked examples

### Cut a tree (consume → modify → create)

```jsonc
"cut_tree": [
  "input.hex.aspect.wood.min: 1",
  "input.slot.0.def_id: corpus",
  "input.has.def_id: axe",
  "output.destroy.slot.0",
  "output.modify.hex.aspect.wood.sub: 1",
  "output.create.slot.0.owner.inventory: corpus-",
  "output.create.slot.0.owner.inventory: log",
  "duration.default: 10",
  "style.default: rtl"
]
```

Action: actor (a corpus card with an axe in their soul-stack) acts
on a tile with `wood >= 1`. On completion: actor dies, tile wood
drops by 1, actor's inventory gains `corpus-` + `log`.

### Plant a sapling (modify add)

```jsonc
"plant_sapling": [
  "input.hex.aspect.wood.min: 0",
  "input.slot.0.def_id: sapling",
  "output.destroy.slot.0",
  "output.modify.hex.aspect.wood.add: 1",
  "duration.default: 5",
  "style.default: rtl"
]
```

Sapling consumed; tile wood rises by 1, saturating at the tree
def's `stock.wood.max`.

### Magnetic success (multi-spawn)

```jsonc
"strike_success": [
  "input.root.def_id: strike",
  "input.slot.0.def_id: corpus",
  "input.slot.1.def_id: corpus",
  "input.slot.2.def_id: corpus",
  "output.destroy.slot.0",
  "output.destroy.slot.1",
  "output.destroy.slot.2",
  "output.destroy.root",
  "output.create.root.inventory: corpus-",
  "output.create.root.inventory: corpus-",
  "output.create.root.inventory: corpus-",
  "duration.default: 10",
  "style.default: ltr"
]
```

## What's intentionally not in v1

- **Tree predicates** (`or.<id>` / `and.<id>` / `not.<...>`). No
  current recipe needs them; adding them later requires the parser
  to step beyond strict left-to-right walk.
- **Card-rooted stock writes** (`output.modify.actor.aspect.X.*`).
  Per-card stock storage doesn't exist yet.
- **Other locations** (`position`, `pocket`, surface literals).
  Grammar reserved.
- **`has_below`** (DOWN-stack predicates) and explicit `has.root.*`.
- **`flag(<ref>, <name>)` modify target** — flag mutation verbs.

## Authoring tips

- One statement per line in the array. Each line is self-describing
  via its path prefix.
- Group by bucket visually (input lines together, output lines
  together) for readability — order doesn't matter to the parser.
- `_comment` keys (`"_comment": "..."`) on outer or inner objects
  are silently skipped — use them sparingly; in-tree doc-comments
  often outlive their accuracy.

## Stable ids

`recipes/id.json` is generated by [`gen-ids.py`](../gen-ids.py). It
keeps a nested `{ "<type>": { "<category>": { "<key>": <int> } } }`
shape (independent of the recipe-data file shape) for stable id
allocation across regenerations. Touch `gen-ids.py` if you change
the outer-key format or category collapses.

`gen-ids.py --skip-known` preserves existing ids and only allocates
fresh ids for new entries; without it, ids re-pack densely from 1.

## Adding a new recipe

1. Add the entry under the appropriate `<type>.<category>` (or bare
   type) key in a `data/*.json` file. Body is a flat statement array.
2. Run `python3 content/gen-ids.py` to allocate a stable id.
3. Run `bin/content test` to verify the recipe parses.
4. Run `bin/st re` to publish into the running shard and seed a
   fresh world.

## Pitfalls

- **Unknown path token** — a misspelled segment will fail at
  registry build with the path verbatim in the error. Check
  spelling and the [Reserved words](#reserved-words) table.
- **Aspect name collision** — an aspect named `min` / `sub` /
  `default` etc. will fail with a "reserved path token" error from
  the aspect-registry build. Rename the aspect.
- **Modify sub > predicate min** — the parser rejects when a
  `sub: N` target shares aspect with the recipe's `hex` predicate
  and `N > min`. Fix by lowering N or raising the predicate min.
- **Slot gap** — if `input.slot.2.*` exists but `slot.1.*` doesn't,
  registry build fails. Always populate slots contiguously.
- **Missing `duration.default`** — required on every recipe.
- **Multiple `input.<anchor>.*` for the same anchor** — registry
  build fails. Each anchor accepts at most one predicate today.
